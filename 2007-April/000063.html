Hi,

Im Zuge der Unterstuetzung von externen Adressbuechern waren leider
noch einige grundlegendere API-Aenderungen an Hibiscus noetig:

1) Ueberweisungen und Lastschriften implementierten das
   Interface "Transfer". Das wurde in "HibiscusTransfer"
   umbenannt

2) Stattdessen gibt es nun ein neues Interface "Transfer",
   welches generisch genug ist, dass es sowohl von HibiscusTransfer
   (und damit von Ueberweisungen und Lastschriften) als auch von
   Umsaetzen implementiert wird.

Hintergrund: Die generische Adressbuch-API liefert Objekte
des Typs "Address". Diese wiederrum haben eine Funktion
"public java.util.List getTransfers()". In dieser Funktion
koennen nun gebuendelt alle Geldbewegungen von und an diese
Adresse zurueckgeliefert werden. Sowohl Ueberweisungen und
Lastschriften als auch Umsaetze.

In dem Zusammenhang wurden 3 Funktionen in "Umsatz" umbenannt.

Bisher:
public String getEmpfaengerName();
public String getEmpfaengerKonto();
public String getEmpfaengerBLZ();

Nun (einheitlich zu Ueberweisungen und Lastschriften):
public String getGegenkontoName();
public String getGegenkontoNummer();
public String getGegenkontoBLZ();

Gruss
Olaf
