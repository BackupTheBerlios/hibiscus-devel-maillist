Hi,

und wieder ein etwas groesseres CVS-Commit ;)
Ich gerade Hibiscus-Erweiterungen eingecheckt,
die es ermoeglichen werden, externe Adressbuecher
anzubinden. Daher werden Adressen nun nicht mehr
direkt aus der Datenbank gelesen sondern ueber
einen neuen Adressbook-Service. Beispiel:

-----------
String suchtext = "blz, kontonummer oder name";

AddressbookService service = (AddressbookService)
   Application.getServiceFactory().lookup(HBCI.class,"addressbook");

GenericIterator found = service.findAddresses(suchtext);
-----------

Der Adressbuch-Service sucht beim Start nach verfuegbaren
Adressbuch-Implementierungen und fragt diese ab.

Falls kein Suchtext angegeben ist, bleibt es der
Adressbuch-Implementierung ueberlassen, ob sie
entweder alle Adressen oder gar keine zurueckliefert.
Das Hibiscus-eigene Adressbuch liefert in dem Fall
alle Adressen. Andere Implementierungen (typischerweise
welche mit grossen Datenmengen) koennten stattdessen
nur dann Daten liefern, wenn ein Suchtext eingegeben
wurde.

Existieren zusaetzliche Adressbuecher, wird der Adressbuch-Dialog
von Hibiscus automatisch um eine Auswahl-Box erweitert,
in der der User entweder in allen oder in einem
speziellen Adressbuch suchen kann.

Weitere Details - unter anderem eine Anleitung, welche
Interfaces man implementieren muss, um eigene Adressbuecher
hinzuzufuegen, werde ich in's Jameica-Wiki schreiben.
