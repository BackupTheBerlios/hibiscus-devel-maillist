>> Mhh, mir faellt da nicht wirklich eine brauchbare technische
>> Loesung ein. Unter Umstaenden koenne ich eine Message schicken,
>> wenn keine Hilfe gefunden wurde. Die koenntest du dann abonnieren
>> und bei dir eine Warnmeldung um Log ausgeben.

> Die Lösung finde ich gut. Dann werde ich entsprechende Loggings nur
> auf meinem Entwicklungssystem ausgeben.

Ist eingebaut. Ich schicke eine QueryMessage an den Channel
"jameica.help.missing". Die QueryMessage enthaelt als "data"
die View, bei der die Hilfe fehlte. Du kannst das also
in etwa so bei dir verwenden:

private class HelpConsumer implements MessageConsumer
{
  public void handleMessage(Message m)
  {
    QueryMessage msg = (QueryMessage) m;
    AbstractView view = (AbstractView) msg.getData();
    Logger.error("help missing for view " + view.getClass());
  }
  [...]
}

MessageConsumer mc = new HelpConsumer();
Application.getMessagingFactory().getMessagingQueue("jameica.help.missing").
  registerMessageConsumer(mc);
